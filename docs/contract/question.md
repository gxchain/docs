## Random number

On the blockchain, a verifiable random number cannot be generated. Generally, a fixed seed is selected to generate, and a random sequence is generated by a hash algorithm such as sha256. Therefore, the safety of random numbers and the selection of seeds are crucial. If the variables available in the chain are selected as seeds, such as the block id, the number of blocks, the balance of the assets, etc., the random algorithm of the contract can be pre-acquired in advance if it is leaked out. Therefore, the random number algorithm should be as complex as possible, or explore the possibility of acquiring out-of-chain data as a random number seed.

- 1. Create a random number for each of the opponents (Alice and Bob) and calculate their hash value.
```bash
Alice:Randomly create a string, for example: "I am Alice" and generate a sha256 hash
zhaoxiangfei@zhaoxiangfeideMacBook-Pro:~$ echo "I am Alice" | shasum -a 256
da161d9ee3c116083030737d5a4d478ee1a7654f8ea51e0513020937a9206b0f

Bob:Randomly create a string, for example: "I am Bob" and generate a sha256 hash
zhaoxiangfei@zhaoxiangfeideMacBook-Pro:~$ echo "I am Bob" | shasum -a 256
77f3bfe06cc926329510b60eba56f80a8d32cf35496587d1444fc690c806c0f9
```
- 2. The first interaction will submit the hash value to the chain.
```
After submitting a hash value to the chain, you cannot reverse the string based on the hash value.
```

- 3. Both parties submit their own random string, and the contract is checked against the submitted string and the hash value of the last commit. Then use two hash values and two random strings for splicing to generate a hash sequence as a random number.
```bash 
sha256("da161d9ee3c116083030737d5a4d478ee1a7654f8ea51e0513020937a9206b0f"+"77f3bfe06cc926329510b60eba56f80a8d32cf35496587d1444fc690c806c0f9"+"I am Alice"+"I am Bob")
```

## Contract and distributed storage

Smart contracts bring the possibility of decentralization of complex business logic, and can store relevant business data in the chain to develop decentralized DAPP. The storage on the chain uses RAM resources, which are suitable for storing high-frequency read and write data. The RAM on the chain is expensive to store. Some large files do not need to be stored in the chain ram. You can use the distributed storage service provided by GXChian to make large files. Storage is hosted on [BaaS Storage](../baas-api/)

## Contract fee 

In order to make rational use of blockchain resources, each time a smart contract is invoked, a certain miner fee needs to be burned. The cost consists of three parts: the base fee (fixed), the memory fee (based on the persistent storage usage), and the CPU fee (according to this The CPU time billing for the next call), the price of the three types of fees and the CPU limit of the calling contract can be dynamically adjusted by the board. Smart contract fee calculation rules:

Contract Deployment Fee: Base Fee + Transaction Message Size * Unit KB Fee

```cpp
deploy_fee = basic_fee+transaction_size*price_per_kb 
```

Contract call fee: Base fee + ram usage + cpu usage

```cpp
transaction_fee = basic_fee + ram_usage * price_per_kb + cpu_usage * price_per_ms
```
